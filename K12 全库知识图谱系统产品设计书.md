K12 全库知识图谱系统 - 产品设计与架构白皮书 (v2.0)

一、 产品定位与核心体验

产品形态：基于边缘计算 (Edge Computing) 的超轻量级、沉浸式 K12 学科知识探索引擎。

设计语言：Godly / Diagram 极简暗黑科技风 (Ultra-minimal Dark Mode)。强调留白、光影、微交互与数据的纯粹性。

核心价值：将碎片化的考点、难点通过点线面的形式重构，帮助学生建立“上帝视角”的学科知识坐标系。

二、 核心技术架构 (Architecture)

为保障极致的加载速度与跨端部署能力，系统采用彻底的 Serverless + Edge 架构：

核心框架：Next.js 15+ (App Router)

运行环境：Cloudflare Edge Runtime (完全抛弃传统 Node.js 限制)

数据库：Cloudflare D1 (Serverless SQLite)

ORM 映射：Prisma Client + @prisma/adapter-d1

动画与手势引擎：Framer Motion (接管拖拽、缩放、路由过渡)

样式方案：Tailwind CSS (极致的原子化 CSS，负责处理毛玻璃与发光特效)

三、 UI/UX 设计规范 (Diagram 风格基调)

极简暗黑基调 (Ultra-minimal Dark)：

背景层：纯黑 #000000 叠加 opacity-3 的白色微缩点阵。

环境光：中央巨大的 blur-[120px] 径向微白光，拉开空间深度。

悬浮胶囊栏 (Floating Pill Toolbar)：

位置：屏幕底部居中，取代传统的侧边栏/顶栏。

材质：#111111 高透明度配合 backdrop-blur-xl，1px 白色渐变边框。

信息降噪卡片 (Noise-cancelled Cards)：

色彩：黑灰阶 (#0A0A0A 到 #1A1A1A)，去除多余的高饱和色。

交互：悬停放大 scale: 1.02，激活时产生微弱的环境发光。

动态神经网络线 (Neural Connections)：

未激活：极低透明度的白色细线 (rgba(255,255,255,0.08))。

激活：选定节点时，相关的关系网络点亮为带阴影的白线。

更新

### 1. 产品定位

打造一个覆盖 12 年制（小学、初中、高中）全学科的自适应学习平台。其核心驱动力是**知识点依赖图谱**，即：如果 A 是 B 的基础，学生必须掌握 A 才能解锁 B。

| **字段名**      | **类型**   | **说明**                             |
| ------------ | -------- | ---------------------------------- |
| `id`         | String   | 唯一标识符（例如：`MATH_G7_001`）            |
| `subject`    | String   | 学科（如：数学、物理、化学）                     |
| `grade`      | String   | 年级（如：七年级、高一）                       |
| `module`     | String   | 章节/模块（如：数与代数、力学）                   |
| `pointName`  | String   | 知识点名称（数据库字段名 `point_name`）         |
| `weight`     | Float    | 考察频率/重要权重 (1.0 - 10.0)             |
| `difficulty` | Int      | 难度等级 (1 - 5)                       |
| `content`    | String?  | Markdown 格式的详细解析（含 LaTeX 公式）       |
| `parents`    | Relation | 前置依赖知识点（即 JSON 中的 `prerequisites`） |

### 3. 全科目数据收集标准 (Standard Operating Procedure)

为了确保“无遗漏”和“模板一致”，AI 生成数据时必须执行以下标准。

#### 3.1 覆盖范围清单

- **数学**：数与代数、方程与不等式、函数（一次/二次/反比例）、几何（三角形/圆/四边形）、统计概率。

- **物理**：声、光、热、力学（浮力/压强）、电学（欧姆定律/电功率）、磁学。

- **化学**：物质构成、空气/水、碳/金属、酸碱盐、溶液。

- **英语**：词法（名词/代词/形容词）、时态（8 大核心时态）、语态、从句（宾语/定语）。

#### 3.2 JSON 生产指令模板 (Prompt)

> 3. *指令：** “请按照我的 `KnowledgePoint` 模型，生成 **[学科名]** 的全考点 JSON。要求： 1. 字段名必须为 `id`, `subject`, `grade`, `module`, `point_name`, `weight`, `difficulty`, `content`, `prerequisites` 。 2. `prerequisites` 数组必须准确指向前置知识点的 `id` 。
>    3. `content` 必须包含核心定义、公式（用 $...$ 包装）和解题关键。”

### 4. 系统功能架构

#### 4.1 后端：自动化同步

- **逻辑**：读取 `data/subjects/` 下所有 JSON，先 `upsert` 基础字段，再遍历建立 `connect` 依赖关系。

- **环境**：基于 Prisma 7 + LibSQL 适配器，支持 Cloudflare Workers 部署。

#### 4.2 前端：可视化技能树

- **展示**：以 SVG 或 Canvas 渲染层级图，父节点在上，子节点在下。

- **交互**：点击节点展示 Markdown 解析；根据 `userProgress` 状态（掌握度）变色。
